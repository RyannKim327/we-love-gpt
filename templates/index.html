We Love GPT API Documentation

POST: /api/chat

Parameter Type Optional/Value
messages JSON Array Required
websearch Boolean Optional

Parameters of messages
role String user | system | assistant
content String Required
curl -X POST "https://{{ url }}/api/chat" \ -H "Content-Type: application/json" \ -d '{ "messages": { "role": "user", "content": "Hello, world!" } }'

GET: /api/chat

Parameter Type Optional/Value
message String Required
websearch Boolean Optional
u String Optional
https://{{ url }}/api/chat?message=hello
 The u parameter defines "UserID" or "Username" which helps to store your data and retrieve anytime. You may response "clear" without any quotation mark as value for message to clear the past messages.

Result

// Success
{
 "status": 200,
 "response": "Hello"
} // Error
{
 "status": 404,
 "response": "Some random message about the error"
}
